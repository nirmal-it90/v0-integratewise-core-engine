## STATE TRANSITIONS

                    login() / signup()
                           ↓
    unauthenticated ──→ authenticated
                           ↓
                    startOnboarding()
                           ↓
                       onboarding
                           ↓
                    completeOnboarding()
                           ↓
                       completed ──→ [Full Access] ──→ logout() → unauthenticated
                           ↓
                       /dashboard
                    /integrations/manager
                    /configuration


## ROUTING GUARDS

Route                      Required Status         Action if Not Met
────────────────────────────────────────────────────────────────────
/                         Any                     Redirect to /landing
/landing                  Any                     Always accessible
/auth/login               Any                     Always accessible
/auth/sign-up             Any                     Always accessible
/onboarding/select        authenticated+          Redirect to /landing
/onboarding               authenticated+          Redirect to /landing
/dashboard                completed               Redirect to /landing
/integrations/manager     completed               Redirect to /landing
/configuration            completed               Redirect to /landing

## PERFORMANCE NOTES

- All operations are synchronous (localStorage)
- No network calls (frontend-only mock)
- Suspense boundaries prevent hydration errors
- useRouter for smooth page transitions
- useContext for global state access
- Progress persists if user abandons flow
