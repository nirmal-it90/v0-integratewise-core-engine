# Database Migrations for Core Engine

This directory contains SQL migration files for the IntegrateWise Core Engine database.

## Database Schema

The core engine uses **Neon Postgres** to store:
- SPINE events (normalized webhook events)
- AI-generated tasks
- Event analytics and insights

## Running Migrations

### Option 1: Using psql
```bash
export DATABASE_URL="postgresql://user:pass@host/db"
psql $DATABASE_URL < migrations/001_create_spine_events.sql
psql $DATABASE_URL < migrations/002_create_ai_tasks.sql
```

### Option 2: Using Neon CLI
```bash
neon sql-exec --file migrations/001_create_spine_events.sql
neon sql-exec --file migrations/002_create_ai_tasks.sql
```

### Option 3: Using Node.js
```bash
cd apps/integratewise-core-engine
node -e "
const { neon } = require('@neondatabase/serverless');
const fs = require('fs');
const sql = neon(process.env.DATABASE_URL);

(async () => {
  const migration1 = fs.readFileSync('migrations/001_create_spine_events.sql', 'utf8');
  const migration2 = fs.readFileSync('migrations/002_create_ai_tasks.sql', 'utf8');
  
  await sql(migration1);
  console.log('✅ Migration 001 applied');
  
  await sql(migration2);
  console.log('✅ Migration 002 applied');
})();
"
```

## Tables

### `spine_events`
Stores all normalized webhook events in SPINE format.

**Columns:**
- `id` (UUID) - Primary key
- `source` (VARCHAR) - Event source (stripe, slack, etc.)
- `type` (VARCHAR) - Event type
- `timestamp` (TIMESTAMPTZ) - Original event timestamp
- `payload` (JSONB) - Event data
- `metadata` (JSONB) - Additional metadata
- `created_at`, `updated_at` (TIMESTAMPTZ)

### `ai_tasks`
Stores tasks generated by AI from events.

**Columns:**
- `id` (UUID) - Primary key
- `event_id` (UUID) - Reference to spine_events
- `workspace_id` (UUID) - Workspace identifier
- `title` (VARCHAR) - Task title
- `description` (TEXT) - Task description
- `priority` (VARCHAR) - low | medium | high | critical
- `status` (VARCHAR) - pending | in_progress | completed | cancelled
- `due_date` (TIMESTAMPTZ) - Optional due date
- `assigned_to` (UUID) - Optional assignee
- `model` (VARCHAR) - AI model used
- `confidence_score` (DECIMAL) - AI confidence (0.00-1.00)
- `metadata` (JSONB) - Additional data
- `created_at`, `updated_at`, `completed_at` (TIMESTAMPTZ)

## Indexes

All tables include appropriate indexes for:
- Primary key lookups
- Source/type filtering
- Time-based queries
- JSONB payload searching (GIN indexes)

## Environment Variables

Required environment variables:
- `DATABASE_URL` - Neon Postgres connection string (pooled)
- `DATABASE_URL_UNPOOLED` - Direct connection string (for migrations)
