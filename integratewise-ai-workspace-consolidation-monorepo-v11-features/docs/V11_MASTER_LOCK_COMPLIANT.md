## B) 5 Core Principles (Corrected)

These five principles are the foundation of the v11.0 system. They govern architecture, UX, and strategic value, ensuring the platform is not a dashboard tool, but compounding intelligence infrastructure.

### 1. Keep Your Tools
IntegrateWise does not replace Slack, Notion, or CRMs. It connects them so your work compounds instead of fragmenting.

### 2. Normalize Once. Render Anywhere (BYOT)
All incoming information is normalized once into canonical models inside the Spine and can be rendered in any View or external template layer without breaking source systems.

### 3. The Spine is the SSOT
All operational and intelligence data is unified into a single source of truth, ensuring every department sees the same reality with controlled visibility.

### 4. IQ Hub Compounds Thinking
AI-generated insights and human notes become durable, searchable, reusable work—so critical knowledge doesn't evaporate into chat history.

### 5. Governance is Intrinsic
Security, auditability, and policy enforcement are built into every stage—from ingestion to write-back—through AI-Relay, Governor Slack, and audit logs.

## C) The System Loop: Load → Store → Think → Act → Govern (Corrected)

The platform runs a five-stage loop to capture, normalize, surface, operationalize, and govern intelligence.

### 1) Load — The Ingestion Layer

Load captures raw signals from:

* Connectors (CRM, calendar, support, docs)
* Dump intake (notes, PDFs, chat exports, markdown)
* Slack triage intake
* Webhooks and tool events

✅ **Important:** Load is not "IQ Hub."  
Load is the intake pipeline feeding normalization.

---

### 2) Store — The Unification Layer (The Spine)

The Spine is the Single Source of Truth (SSOT). All captured data is normalized into canonical entities (Account, Deal, Task, Interaction, HealthScore, etc.) with provenance and auditability.

---

### 3) Think — The Intelligence Layer (IQ Hub)

IQ Hub is the user-facing working memory where:

* captured knowledge is structured into reusable assets
* strategies, templates, playbooks, and decisions are linked to Spine entities
* Cognitive Twin retrieves the right memory at the moment of need

✅ IQ Hub = Think  
✅ Brainstorming Layer / Neutron = internal alias (not primary term)

---

### 4) Act — The Delivery Layer (Views + Safe Actions)

Act is where intelligence becomes usable inside workflows:

* insights are delivered inside role-based Views (CS, Sales, Marketing, PM, BOS, Admin)
* suggested actions and drafts are generated
* safe write-backs are proposed (not blindly executed)

---

### 5) Govern — The Trust Layer

Governance is the platform's safety envelope:

* **Governor Slack (Slack Triage Center):** real-time oversight + approvals
* **AI-Relay Gateway:** single audited gateway for all AI calls + integrations
* **Audit Trail (AuditLog):** immutable history of every action
* **Policy Gates:** RBAC, PII controls, write-back restrictions

✅ This keeps the system enterprise deployable.

## D) End-to-End User Flow (Corrected)

### 1) First-Time Login (Onboarding) — v11.0 compliant

Onboarding must be frictionless and cannot force heavy setup.

1. **Quick Login**
   Login with Google / Microsoft / Email.

2. **Persona Analysis (6–10 seconds)**
   Role + intent signals are used only to avoid a blank start.

3. **Persona Insights**
   You get a practical recommended default View/Lens (confirm or override).

4. **Load Your Data (Connect / Dump / Skip)**

* Connect 1–2 tools
* Dump files / notes
* Skip with demo data

✅ No "Invite team mandatory"  
✅ No "Connect everything mandatory"  
✅ No "heavy setup required"

---

### 2) Post-Login Daily Loop

1. **Login → Today View (default)**
2. **Navigate into the right View (CS/Sales/Marketing/PM/BOS/Admin)**
3. **Cognitive Twin surfaces prepared context + recommended actions**
4. **IQ Hub captures and structures new thinking**
5. **Governance records + approves sensitive actions via Slack + AI-Relay**
